# Personal Value Mining

## Core Principle

Mine for evidence before writing positioning. A good personal page is not flattering copy; it is compressed proof.

In this project, extracted evidence must be saved as Markdown under `source-data/`. The rendered page and agent files should transform that evidence, not become new data stores.

## Workflow

1. Identify the primary audience.
2. Gather evidence from public work, private notes with permission, and direct answers.
3. Group evidence by what it proves, not by source type.
4. Write one value claim.
5. Build proof clusters with "what this proves" sentences.
6. Add a working-with-me section.
7. Store the result in `source-data/*.md` and produce both human-readable and AI-readable outputs from it.

## Evidence Categories

- Shipped products and prototypes
- Writing and public thinking
- Community or team facilitation
- Research and decision documents
- Reusable templates, scripts, and systems
- Repeated taste patterns or constraints

## Quality Bar

Every strong claim needs one of:

- Public link
- Specific project
- Concrete artifact
- User/community outcome
- Explicit owner-provided context

## Common Mistakes

- Starting with adjectives like "creative" or "strategic".
- Listing everything instead of curating proof.
- Copying another person's positioning.
- Treating GitHub stars as the only signal.
- Ignoring private but permissioned context that explains the real through-line.
- Leaving the real evidence only inside generated page code.
